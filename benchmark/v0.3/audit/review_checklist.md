# Gold audit checklist (v0.3)

This checklist governs **human** review of obligations, semantic units, and
behavioral harnesses for benchmark `v0.3`. It applies to instances listed in
`splits/eval.json` (the eval split used for paper-facing claims).

## Repository posture (`gold_signoff.json`)

`v0.3` ships with `release_gold_audit_status: "template_pending_human_review"`.
In that posture:

- `approved` **must** remain `false`.
- `primary_reviewer` / `secondary_reviewer` are **slots** (the file may say
  `Unassigned` until real names are recorded).
- Evidence CSVs under `audit/evidence/` contain **skeleton rows** (`status=pending`,
  empty verdict columns) generated by `cta benchmark audit-workbook`.

That posture is **honest**: the tree is automation-ready, not claiming a
finished dual-pass audit. When both human reviewers complete the workbook and
agree, curators must:

1. Fill `evidence/per_instance_audit.csv` and `evidence/obligation_audit.csv`.
2. Record closing rationale in `evidence/signoff_notes.md`.
3. Set `release_gold_audit_status` to `human_audit_complete` (or remove the
   field, which defaults to the strict legacy posture), set **real** reviewer
   names, and set `"approved": true`.

Release validation (`cargo run -p cta_cli -- validate benchmark --version v0.3 --release`)
accepts **either**:

- **Template posture**: `template_pending_human_review` with `approved: false`, or
- **Complete posture**: (field omitted or `human_audit_complete`) with two
  non-empty reviewer names and `approved: true`.

## What reviewers confirm (per instance)

- Every critical semantic unit is linked to at least one gold obligation.
- No gold obligation is vacuous.
- No gloss overstates the Lean statement.
- No precondition is encoded as a universal truth.
- No critical property is represented only weakly or ambiguously.
- Scaffold obligations and gold obligations are semantically aligned.
- The behavioral harness is compatible with intended semantics.

## Signoff protocol

1. Primary reviewer completes first pass in `evidence/per_instance_audit.csv`
   and `evidence/obligation_audit.csv`.
2. Secondary reviewer performs an independent pass on the same rows.
3. Disagreements are resolved and disposition is recorded in
   `evidence/obligation_audit.csv`.
4. Final notes are captured in `evidence/signoff_notes.md`.
5. `gold_signoff.json` is updated: real reviewer names, `approved: true`, and
   `release_gold_audit_status` set to `human_audit_complete` (or field removed
   after audit), **only** after both reviewers agree.

## Regenerating skeleton workbooks

From the repository root:

```bash
cargo run -p cta_cli -- benchmark audit-workbook --version v0.3
```

This refreshes skeleton CSVs from `splits/eval.json` and each instance’s
`reference_obligations.json`. **Do not** regenerate after reviewers have filled
rows unless you first archive their work.
